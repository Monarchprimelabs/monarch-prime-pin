# Monarch Prime PIN Agent Rules

- Do not rewrite the app unless explicitly approved.
- Do not delete working features to simplify the codebase.
- Do not rename routes, screens, bundle identifiers, entitlement keys, product IDs, or storage keys unless necessary and approved.
- Do not add or expand auth, cloud sync, database, notification, or subscription logic unless explicitly approved.
- Do not commit secrets, signing files, provisioning files, local archives, or generated folders.
- Work in branches, not directly on `main`.
- Open a pull request for every meaningful change.
- Run the documented checks before saying work is done.
- Treat this app as production-sensitive. Preserve release behavior, legal copy, App Store assumptions, and monetization flow unless explicitly approved.
- Update the docs when release assumptions, setup, or workflow rules change.
